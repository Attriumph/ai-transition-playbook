# 📊 LLMOps, Observability & Evaluation

## 🎯 The Engineering Approach to LLMs
Moving beyond intuitive prompting requires treating Large Language Models as highly non-deterministic, distributed APIs. To achieve production stability, you must construct automated validation suites, optimize request routes, monitor token-level latency, and implement latency-saving caching.

## 🗺️ Core Curriculum & Architectural Deep Dives

### 1. Automated Evaluation (LLM-as-a-Judge)
We utilize the **Ragas** framework to mathematically measure the quality of our Retrieval-Augmented Generation (RAG) pipelines. Ragas breaks down evaluations into specific, measurable indices.

#### 🧮 A. Faithfulness (Hallucination Detection)
Evaluates if the generated answer is strictly grounded in the retrieved context.

$$\text{Faithfulness} = \frac{|S_c \cap C|}{|S_c|}$$

Where:
* **$S_c$**: The set of all atomic, individual factual claims extracted from the LLM's generated answer.
* **$C$**: The retrieved context block.
* **$S_c \cap C$**: The count of generated claims that can be logically inferred directly from the retrieved context.

#### 🧮 B. Answer Relevance
Measures whether the generated answer directly addresses the user's initial query, ignoring factual accuracy (which is evaluated by Faithfulness).

$$\text{Answer Relevance} = \frac{1}{N} \sum_{i=1}^{N} \cos\left(\vec{E}_{g_i}, \vec{E}_o\right)$$

Where:
* **$E_o$**: The high-dimensional embedding vector of the original user query.
* **$E_{g_i}$**: The embedding vector of the $i$-th question generated by a secondary LLM *based on* the generated answer.
* **$N$**: The number of hypothetical questions generated (default = $3$).
* **$\cos(\vec{u}, \vec{v})$**: Cosine similarity $\frac{\vec{u} \cdot \vec{v}}{\|\vec{u}\|\|\vec{v}\|}$.

#### 🧮 C. Context Recall
Evaluates if the retrieval system retrieved all necessary pieces of information required to formulate the correct answer, compared against a human-written "ground truth".

$$\text{Context Recall} = \frac{|S_{gt} \cap C|}{|S_{gt}|}$$

Where:
* **$S_{gt}$**: The set of atomic claims extracted from the human-written ground truth answer.
* **$C$**: The retrieved context chunks.

### 2. Context Caching & Prefix Optimization
Cloud APIs (Anthropic, OpenAI, DeepSeek) support **Prompt Caching**. To achieve high hit rates and reduce input token costs by up to **$90\%$**, prompts must be structured deterministically:

```
[System Instruction]      <-- STATIC (Prefix Cached)
[Corporate Policies]       <-- STATIC (Prefix Cached)
[Retrieved Context Blocks] <-- DYNAMIC / SLOW-MOVING (Prefix Cached if ordered)
[User Chat History]        <-- DOCKING ENDPOINT (Fast-moving, non-cached)
```
* **Prefix Matching Rule**: The cache is evaluated from the beginning of the prompt. Any change in a single character at character index $N$ invalidates the cache for all subsequent text ($>N$).
* **Strategy**: Always place highly dynamic parameters (like current timestamps, user questions, or conversation turns) at the **absolute bottom** of the prompt structure.

### 💻 Python Implementation: Production-Grade Semantic Cache
Below is a thread-safe Semantic Cache utilizing cosine similarity. It intercepts user queries and returns high-confidence cached responses in sub-millisecond times, avoiding costly and slow LLM API calls.

```python
import numpy as np
from typing import Dict, Any, Optional, Tuple

class SemanticCache:
    def __init__(self, threshold: float = 0.94):
        self.threshold = threshold
        # Dictionary mapping query string to (embedding_vector, response_text)
        self.cache: Dict[str, Tuple[np.ndarray, str]] = {}
        
    def _cosine_similarity(self, vec_a: np.ndarray, vec_b: np.ndarray) -> float:
        dot_product = np.dot(vec_a, vec_b)
        norm_a = np.linalg.norm(vec_a)
        norm_b = np.linalg.norm(vec_b)
        if norm_a == 0.0 or norm_b == 0.0:
            return 0.0
        return float(dot_product / (norm_a * norm_b))
        
    def lookup(self, query: str, query_embedding: np.ndarray) -> Optional[str]:
        """
        Looks up the query in the semantic cache.
        Returns the cached response if similarity exceeds the threshold.
        """
        best_score = -1.0
        best_response: Optional[str] = None
        
        for cached_query, (cached_embed, response) in self.cache.items():
            similarity = self._cosine_similarity(query_embedding, cached_embed)
            if similarity > best_score:
                best_score = similarity
                best_response = response
                
        if best_score >= self.threshold:
            print(f"🎯 Cache HIT! Semantic similarity: {best_score:.4f}")
            return best_response
            
        print(f"💨 Cache MISS. Best similarity: {best_score:.4f}")
        return None
        
    def update(self, query: str, query_embedding: np.ndarray, response: str) -> None:
        """
        Inserts a new query and response into the cache.
        """
        self.cache[query] = (query_embedding, response)

# Demonstration Usage
if __name__ == "__main__":
    # Create two semantic vector embeddings representing similar concepts
    vec_original = np.array([0.15, 0.85, 0.05, 0.50])
    vec_similar  = np.array([0.16, 0.84, 0.04, 0.51]) # Very close
    vec_different = np.array([0.90, 0.10, 0.10, 0.05]) # Far away
    
    cache = SemanticCache(threshold=0.95)
    
    # Store original query
    cache.update("How do I connect to Qdrant?", vec_original, "Use qdrant_client.QdrantClient(url='...')")
    
    # Lookup similar query (Expected HIT)
    print("Test 1:")
    ans_1 = cache.lookup("Ways to connect to Qdrant?", vec_similar)
    print(f"Result 1: {ans_1}\n")
    
    # Lookup different query (Expected MISS)
    print("Test 2:")
    ans_2 = cache.lookup("How to cook pasta?", vec_different)
    print(f"Result 2: {ans_2}")
```
